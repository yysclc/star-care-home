# DEV_PLAN.md — 第一阶段灰盒原型开发计划

> 本文档基于 `AI_REQUIREMENTS.md`，将需求收束至第一阶段可实现的灰盒原型范围。
> 目标：跑通"白天 → 夜晚 → 次日白天"完整闭环，验证核心玩法节奏。

---

## 1. 第一阶段玩法目标

- 玩家扮演刚入职的心理学毕业生，完成**2 天**照护所工作体验。
- 白天：在照护所场景中自由移动，与主任、3 名孩子、设施交互，消耗精力行动点。
- 夜晚（仅 Day 1 结束后触发）：进入**主线孩子"小橙"的梦境**，探索记忆碎片，获得 1 张线索卡。
- Day 2 白天：线索卡解锁针对小橙的新互动选项，完成后触发**阶段性结尾**（简短文字总结，不做评分/结局分支）。
- 全程美术使用色块 + 占位文字，不制作正式素材。

## 2. 白天场景对象

### 2.1 场景布局（单屏/分区，占位色块）

```
┌─────────────────────────────────────┐
│  [办公室区]        [感统室区]         │
│   主任 NPC          设施：感统器材    │
├──────────────┬──────────────────────┤
│  [活动室区]   │   [休息角区]          │
│  小橙（主线） │   小木（支线）        │
│  小雨（支线） │                      │
└──────────────┴──────────────────────┘
         [走廊/状态栏 HUD]
```

### 2.2 场景对象清单

| 对象 | 类型 | 说明 |
|---|---|---|
| **玩家角色** | 可移动 | WASD/方向键移动，靠近触发交互提示 |
| **主任** | NPC | Day 1 触发新手引导；每天开始时给出当日任务提示 |
| **小橙** | 主线孩子 | 声音敏感特征；Day 1 有情绪事件，夜晚梦境入口 |
| **小木** | 支线孩子 | 规律依赖特征；有 2–3 条简短互动和日常状态变化 |
| **小雨** | 支线孩子 | 视觉寻求特征；有 2–3 条简短互动和日常状态变化 |
| **感统器材** | 设施 | 可使用，降低附近孩子压力（消耗 1 精力） |
| **告示板** | 设施 | 显示当日任务、机构公告（纯文字占位） |
| **HUD** | UI 层 | 显示：当前天数、精力剩余、资金、口碑数值 |

### 2.3 孩子状态与互动规则

每名孩子每天有独立状态机：**平静 → 焦虑 → 崩溃**

| 互动选项 | 效果 |
|---|---|
| 陪伴（坐在旁边）| 压力 -5，消耗 1 精力 |
| 结构化活动 | 压力 -10，安全感 +5，消耗 2 精力 |
| 感官安抚（使用器材）| 压力 -15，消耗 1 精力，需靠近感统室 |
| 强行要求参与 | 压力 +20，信任 -10（错误选项，有后果提示）|

- **崩溃事件**：压力 ≥ 80 时触发，口碑 -5，强制消耗 2 精力处理。
- **小橙专属**（Day 1）：午后随机触发"突发噪音事件"，是当晚入梦的剧情触发点。

## 3. 夜晚梦境对象（仅 Day 1 结束后，主线孩子小橙）

### 3.1 梦境场景布局（俯视自由移动，有限视野光晕）

```
 起点（小橙的床）
    │
    ├── [家庭客厅区] — 记忆碎片 A：电视噪音（可点击）
    │                  噪声区域（绕行或找到安抚物）
    ├── [学校走廊区] — 记忆碎片 B：下课铃声（可点击）
    │                  记忆碎片 C：同学聚拢对话（可点击）
    └── [安静角落]   — 安抚物：耳机（触碰后视野扩大）
                       理解关键词浮现（收集 2/3 碎片后解锁）
```

### 3.2 梦境对象清单

| 对象 | 类型 | 说明 |
|---|---|---|
| **玩家视野光晕** | 遮罩 | 半径 80px，黑色遮罩 + 中心亮区，象征感官局限 |
| **记忆碎片 A/B/C** | 可点击物 | 点击后弹出 1–2 行文字描述；共 3 个，需收集 ≥ 2 个 |
| **噪声区域** | 障碍区 | 进入后触发屏幕抖动 + 文字提示，压迫感演出 |
| **安抚物（耳机）** | 可拾取 | 拾取后视野半径扩大至 160px；允许穿越噪声区域 |
| **理解关键词** | 触发条件 | 收集 ≥ 2 碎片后在安静角落显示："他需要预告，不需要突然开始" |
| **线索卡弹窗** | 结算 UI | 梦境结束时展示，内容："小橙对突发声音非常敏感，提前告知活动开始时间能帮助他做好准备。" |

## 4. 全局状态字段（GameState）

```js
GameState = {
  // 进度
  day: 1,                    // 当前天数（1 或 2）
  phase: "day",              // "day" | "night" | "transition"

  // 老师数值
  energy: 10,                // 0–10，每天重置为 10

  // 机构数值
  funds: 5000,               // 数值型，每天结算
  reputation: 60,            // 0–100

  // 孩子状态（每名孩子一个对象）
  children: {
    xiaocheng: { stress: 30, safety: 40, trust: 30, state: "calm" },
    xiaomu:    { stress: 20, safety: 50, trust: 40, state: "calm" },
    xiaoyu:    { stress: 25, safety: 35, trust: 35, state: "calm" }
  },

  // 线索卡
  clues: [],                 // 收集到的线索卡 id 数组

  // 标志位
  flags: {
    day1NoiseEventTriggered: false,   // 小橙噪音事件是否触发
    day1DreamCompleted: false,        // Day1 梦境是否完成
    xiaochengNewOptionUnlocked: false // Day2 新互动选项是否解锁
  }
}
```

> `GameState` 在每次关键操作后通过 `localStorage.setItem('asd_save', JSON.stringify(GameState))` 持久化。

## 5. 完整流程（Day 1 → 夜晚 → Day 2）

```
[标题画面 / 序幕]
  玩家角色背景交代（失业、梦见引路者动物、收到录用通知）
  → 进入照护所第一天

[DAY 1 白天]
  1. 主任触发新手引导（走廊对话，说明操作方式）
  2. 玩家自由行动，精力 = 10
     - 可与小橙、小木、小雨交互
     - 可使用感统器材
  3. 午后固定触发「小橙噪音事件」
     （无论玩家是否靠近，旁白弹窗提示小橙捂耳尖叫）
     - 正确应对：选"移到安静角落 + 提前告知" → 压力下降，标记 day1NoiseEventTriggered
     - 错误应对：选"要求他安静" → 崩溃，口碑 -5
  4. 精力耗尽或玩家点击"结束今天" → 进入结算
     - 资金 -200（日常支出），口碑随口碑事件微调
     - 弹出"今天结束了……"过渡文字

[DAY 1 夜晚]
  1. 过渡动画：屏幕变暗，标题"小橙的梦"淡入
  2. 玩家以小橙视角进入梦境场景，有限视野
  3. 自由探索，收集记忆碎片（A/B/C 中至少 2 个）
  4. 找到耳机安抚物（可选，扩大视野）
  5. 收集满足条件后，安静角落出现理解关键词
  6. 点击关键词 → 线索卡弹窗 → 梦境结束，标记 day1DreamCompleted = true

[DAY 2 白天]
  1. 开场旁白："你睡得不太好，却感觉好像明白了一点什么。"
  2. 小橙互动选项新增：「提前告知活动开始时间」（需持有线索卡）
     - 使用后压力 -20，信任 +15，触发小橙的简短积极回应文字
  3. 小木、小雨继续日常互动（各 1 次随机日常事件）
  4. 精力耗尽或玩家点击"结束今天" → 阶段性结尾

[阶段性结尾]
  - 全屏文字总结（不评分，不分支）：
    "两天过去了。小橙今天安静地坐完了整个绘画课——
     也许不是因为你做了什么神奇的事，
     而是因为你开始尝试去理解他。"
  - 显示科普卡片：ASD 是神经发育差异而非疾病……
  - 提示："更多故事，敬请期待。"
```

## 6. 文件与模块结构

```
/src
  main.js                  # Vite 入口，初始化 Phaser Game

  /scenes
    BootScene.js           # 预加载占位资源（色块纹理、字体）
    PrologueScene.js       # 序幕：玩家背景 + 梦见引路者 + 录用通知
    DayScene.js            # 白天照护所主场景（Day 1 & Day 2 复用）
    NightScene.js          # 夜晚梦境场景（Day 1 结束后触发）
    EndingScene.js         # 阶段性结尾 + 科普卡片

  /systems
    GameState.js           # 全局状态定义 + save/load (localStorage)
    EnergySystem.js        # 精力消耗与行动点管理
    StressSystem.js        # 孩子压力/状态机（平静/焦虑/崩溃）
    EventBus.js            # 场景间事件通信

  /data
    children.js            # 3 名孩子的初始属性、互动选项、对话文本
    events.js              # 日常事件表（噪音事件、日常事件等）
    clues.js               # 线索卡内容定义

  /ui
    HUD.js                 # 顶部状态栏（天数/精力/资金/口碑）
    DialogBox.js           # 对话/旁白弹窗组件
    ClueCard.js            # 线索卡展示弹窗
    ChoicePanel.js         # 互动选项面板（2–4 个选项按钮）

  /objects
    Player.js              # 玩家角色（移动、碰撞、交互范围）
    NPC.js                 # 通用 NPC 类（主任、孩子）
    Interactable.js        # 可交互物体基类（设施、梦境碎片、安抚物）
    FogOfWar.js            # 夜晚视野遮罩（光晕效果）
```
