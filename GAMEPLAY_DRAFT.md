# GAMEPLAY_DRAFT.md — 星星照护所·世界观与玩法设计草案

本草案旨在明确《星星照护所》的世界观设定，提炼叙事缺口，并规划教师办公室及地图场景的玩法候选方案。

---

## 一、 世界观与叙事设定

### 1.1 基础设定提炼

*   **星星照护所**：一家为自闭症谱系（ASD）儿童提供日间照护的小型公益机构。目前照护一些患有ASD谱系障碍的孩子。机构面临资金紧张、家长信任波动、社会口碑亟待维护等多重现实压力。
*   **命名由来**："自闭症，来自星星的孩子"——这是一个由来已久的民间隐喻，也是照护所的精神内核：每个孩子都有属于自己的发光方式，只是频率不同。
*   **主角**：心理学背景的刚入职实习工作人员，性格热情、爱学习。最初对ASD儿童的了解仅来自书本知识。入行起点轻松：长时间找不到心仪工作的心理学毕业生，意外收到远方表亲的一封信（甚至是飞鸽传信），就此来到照护所。真正入行的私人动因留待后续揭秘。通过日常的"观察、记录、学习"，体现出愿意倾听、谨慎尝试、重视建立长期信任的成长弧线。
*   **关系网**：
    *   **孩子**：个体的感知和表达方式不同，信任需要通过专业且细致的陪伴逐渐积累。
    *   **家长**：焦虑的主要来源，承受着社会偏见的孤立感和对孩子进展的期盼，其信任度直接挂钩机构的留存。每天结束后可进入所外视角，呈现家长的现实困境。
    *   **所长陈岚**：经验丰富的引导者、撑伞人。在日常中提供专业支持，在危机中给予决策保障。后续可揭示：陈岚本人可能是神经多样性者（如阿斯伯格），曾是全家眼中的"怪人"，这一背景构成他坚守照护所的内在动力与剧情反转。
    *   **公众/社会**：口碑决定了线上筹款效率，反映大众对自闭症及照护机构的认知与接纳程度。

### 1.2 世界观缺口（已确认方向）

1.  **命名故事**：已确认——"星星照护所"源自"来自星星的孩子"这一隐喻，无需进一步开发。
2.  **所长坚持的动机**：陈岚本人可能是神经多样性者，曾是"怪人"，这段个人史是照护所存在的真正情感根基，留作中后期剧情反转。
3.  **主角的连接**：入职起点轻松（找不到工作 + 意外来信），私人深层原因留作后续揭秘，避免开头设定过重。
4.  **家长焦虑的具象化**：每天游戏结束后加入"所外视角"短剧情，呈现家长接孩子回家后的现实困境，而非只在机构内用数值体现。
5.  **经营指标的联动**：资金枯竭或口碑崩溃不过度设计，设2-3个节点即可。第一次出现预警，所长出手提升数值；第二次触发游戏结束。
6.  **孩子的家庭背景**：暂时仅围绕主要角色橙橙展开（见第三章）。
7.  **机构日常节奏**：目前重点刻画所长与主角的互动，其他工作人员作为背景存在，不单独展开剧情线。

| **6. 茶水间** | 冲咖啡榨取体力，有概率带来次日体力上限折损 | ¥20 | 体力 +1；30% 概率次日上限 -1 | **第二阶段**（先不加） |

### 2.2 电脑入口：功能设计

电脑集成**经营、学习、对外沟通**三类功能，是 Hub 的核心操作中心。

---

**① 整理今日照护记录（硬性任务）**

- *前置条件*：今日已与至少一名孩子进行过互动。未满足时点击，系统提示"今天还没有互动记录，去看看孩子们吧"，不消耗体力，不触发惩罚。
- *消耗/产出*：10 体力 → 任务完成标志。
- *适合第一版*：是。

---

**② 公益直播与线上筹款**

- *机制*：发起直播，筹得资金。产出由**玩家属性**和**照护所名望**共同决定，分为以下档位：

| 条件 | 结果 |
| :--- | :--- |
| 专业理解 ≥ 3，名望 ≥ 60 | 高收益，+¥500，名望小幅提升 |
| 专业理解 ≥ 2，名望 30–60 | 正常收益，+¥200 |
| 专业理解 < 2，任意名望 | 说错词翻车，资金 +0，名望 -10 |
| 任意属性，名望 < 30 | 直播被质疑，50% 概率触发舆论事件，名望额外 -5 |

- *消耗*：5 体力。
- *适合第一版*：是（先实现2档，后续扩展到4档）。

---

**③ 购买设施**

- *机制*：采购辅助设施（降噪耳机、视觉日程卡、软垫等），购买后解锁对应场景的被动效果或新互动选项。
- *消耗*：¥300–¥600/件，资金一次性扣除。
- *风险*：资金不足时无法购买；购买后若资金见底，遇突发事件无法应对，可能触发经营危机节点。
- *适合第一版*：是。

---

**④ 学习 ASD 专业课程**

- *机制*：在线进修特殊教育课程。
- *消耗*：5 体力 + ¥150。
- *产出*：+1 专业理解（或 +1 沟通能力，视课程类型）。
- *风险*：属长期投资，若当日硬性任务未完成、体力已不足，盲目学习可能导致当天经营失败。
- *适合第一版*：是。

---

**⑤ 家长沟通（整合"电话/聊天软件/回复信件"）**

- *呈现方式*：电脑内的"消息应用"，整合文字消息、语音留言（文字呈现）、邮件三种形式，根据剧情阶段触发不同类型。
- *消耗*：5 体力/次。
- *机制*：选择题形式——玩家需在 2–3 个回应选项中选择，结果由回应质量和**沟通能力属性**共同判定：

| 回应质量 | 判定条件 | 结果 |
| :--- | :--- | :--- |
| 同理心 + 专业表达 | 选项正确 + 沟通能力 ≥ 2 | 家长信任 +，名望 + |
| 正确选项但能力不足 | 选项正确 + 沟通能力 < 2 | 家长信任小幅 +，无名望加成（"听起来挺有道理但不够专业"） |
| 否定/治愈叙事 | 错误选项 | 家长信任 -，名望 - |
| 联系媒体/回应投诉 | 特定事件触发 | 数值变化与**专业理解 + 名望**挂钩，可高收益也可翻车 |

- *适合第一版*：部分（先做"家长日常咨询"和"投诉应对"两类，联系媒体放第二阶段）。

## UI 与属性术语说明

### 照护所属性

照护所属性是机构层面的资源与评价，和玩家主控的个人能力分开管理。

| 属性       | 说明                                 | 显示位置                    |
| -------- | ---------------------------------- | ----------------------- |
| **金钱**   | 照护所运营资金。用于日常支出、修建房间、购买设施和应对突发经营事件。 | 顶部 HUD 固定显示             |
| **名望**   | 照护所在社会层面的口碑与认知度，影响筹款效率、舆论事件和部分结局。  | 顶部 HUD 固定显示             |
| **家长信任** | 家长群体对照护所的整体信任。与普通孩子群体状态和家长投诉概率挂钩。  | 电脑 → 家长沟通页面查看，不放入顶部 HUD |

### 玩家主控个人属性

玩家主控属性代表主角个人能力成长。

| 属性       | 说明                        | 影响方向                       |
| -------- | ------------------------- | -------------------------- |
| **专业理解** | 对 ASD、照护支持方式、儿童行为观察的理解能力。 | 影响照护记录质量、课程学习、部分互动选项和公益表达。 |
| **沟通能力** | 与家长、所长、社会公众和孩子进行有效沟通的能力。  | 影响家长信任、公益直播、聊天回复、危机解释。     |
| **体能表现** | 主控的长期身体状态。                | 影响行动力上限，后续也可影响高消耗活动的成功率。   |

### 行动力

行动力是每日行动资源，和体能表现分开。

* **行动力**：当前一天里还能行动多少次。自由行动按钮通常消耗 1 点行动力；固定活动根据活动强度消耗不同数量的行动力。
* **体能表现**：长期属性。体能表现提升后，行动力上限提升。
* 采用规则：体能表现每增加 10，行动力上限增加 2。
* 行动力每天开始时恢复到当前上限。

## 孩子数值系统

### 普通孩子群体

除橙橙外，其他孩子暂时不做单独角色线，统一使用两个群体数值表示整体状态。

| 数值        | 说明                                |
| --------- | --------------------------------- |
| **群体信任度** | 普通孩子对照护所、活动安排和主控的整体接受程度。范围 0–100。 |
| **群体压力值** | 普通孩子当前整体情绪负荷。范围 0–100。            |

普通孩子群体状态与家长信任挂钩。

当：

```text id="m77mda"
群体压力值 > 群体信任度
```

时，家长投诉概率开始线性增加。

候选规则：

```text id="z4f8yi"
投诉风险 = 群体压力值 - 群体信任度
```

* 投诉风险 ≤ 0：无投诉风险；
* 投诉风险 1–10：低风险；
* 投诉风险 11–25：中风险；
* 投诉风险 26 以上：高风险；
* 名望高时可降低风险等级；
* 名望低时可提高风险等级。

如果需要百分比实现，可使用简化公式：

```text id="p6yud5"
投诉概率 = clamp((群体压力值 - 群体信任度) × 1.5%, 0%, 60%)
```

名望可作为修正项，但第一版可以先用风险等级，避免数值系统过细。

### 橙橙

橙橙是主线孩子，不进入普通孩子群体数值系统。

橙橙只使用一个长期数值：

| 数值        | 说明                           |
| --------- | ---------------------------- |
| **橙橙好感值** | 橙橙对主控的信任、接受和关系推进程度。范围 0–100。 |

橙橙好感值用于：

* 解锁橙橙个人剧情；
* 决定橙橙是否愿意让玩家靠近她的表达方式；
* 影响阶段结局和最终结局；
* 玩家选择错误时降低好感值；
* 好感值不足时，不解锁后续剧情。

橙橙不单独设置压力值、安全感值或崩溃状态系统。
橙橙相关剧情通过好感值、剧情 flag 和玩家选择结果推进。现阶段先跑通流程，橙橙个人剧情作为额外策划案后续单独细化。

### 橙橙剧情的代码分割建议

橙橙个人剧情适合单独从主系统中分离出来。

后续实现时可以考虑单独设置：

```text id="b9it2v"
data/chengchengStory.js
```

用于存放：

* 橙橙个人剧情节点；
* 好感值阈值；
* 已解锁剧情 flag；
* 绘画相关文本；
* 玩家选择与好感变化；
* 剧情是否已播放。

这样主玩法系统只负责好感值和触发条件，具体剧情内容不混进地图、固定活动或办公室主逻辑里。
当前阶段先不写具体剧情，只保留代码分割建议。

## 固定行动阶段与自由行动阶段

一天的行动阶段分为两类：固定行动阶段和自由行动阶段。

### 固定行动阶段

每天从办公室开始。除第一天外，每天开始时会出现：

```text id="c5mq4m"
去上课 / 组织活动
```

的选项。

点击后进入固定行动阶段。

固定行动阶段的含义是：给全照护所小朋友统一安排活动。

一天中必须完成三次固定行动，分别对应：

* 早上第一次；
* 早上第二次；
* 下午一次。

三次固定行动全部完成后，才进入自由行动阶段。

固定行动可以重复选择同一场景，但重复会产生惩罚：

* 第一次安排：正常收益；
* 第二次重复：收益下降，压力变化变差；
* 第三次重复：信任收益很低，压力明显上升，可能触发小事件。

固定行动主要影响普通孩子群体数值：

* 群体信任度；
* 群体压力值。

目前只有橙橙是单独计算的主线孩子，其他孩子统一进入普通孩子群体系统。

固定行动基础可选场景：

* 活动室；
* 娱乐 / 玩具室；
* 户外探索院。

其余场景需要通过电脑修建后开放：

* 绘画室；
* 感统室；
* 图书室。

其中，**绘画室不是初始开放场景，但第一天就可以修建，且修建后当天立刻可用**。玩家在第一天观察橙橙后，可以意识到绘画空间的重要性，并选择投入资金修建绘画室。修建本身也是主角观察与行动之间的第一次连接。

每个固定活动场景需要设定：

* 对群体信任度的影响；
* 对群体压力值的影响；
* 重复安排同一活动的惩罚；
* 消耗多少行动力；
* 是否可接小游戏；
* 小游戏得分如何影响活动效果。

第一版小游戏完全占位，不实际实现。
小游戏未来应作为单独模块实现，不直接混入地图场景主代码。后续可考虑单独拆分为：

```text id="eov5d5"
minigames/
  scheduleActivityGame.js
  drawingActivityGame.js
  sensoryRoomGame.js
```

当前阶段只在文档中保留“小游戏得分影响活动效果”的设计接口。

### 固定活动的行动力消耗原则

固定活动不是统一消耗 1 行动力，而是和活动能带来的增益、组织难度和体能消耗相关。

建议规则：

| 活动类型          | 行动力消耗 | 设计理由                             |
| ------------- | ----: | -------------------------------- |
| 低强度、低风险活动     |     1 | 如活动室卡片活动、图书室共读，组织成本低。            |
| 中等强度、较高收益活动   |     2 | 如绘画室、玩具室，能提升信任或降低压力，但需要更多照看。     |
| 高体力 / 高调节收益活动 |     3 | 如户外探索院、感统室，可能带来明显减压或体能成长，但组织成本高。 |

固定活动的小游戏分数后续可影响该次活动收益：

* 分数高：增强正面效果；
* 分数中：保持基础效果；
* 分数低：降低收益，或增加压力惩罚。

### 自由行动阶段

固定行动阶段结束后进入自由行动阶段。

自由行动阶段中，玩家可以进入已开放房间，点击房间中的具体交互按钮。

自由行动主要用于：

* 提升玩家个人属性；
* 提升照护所属性；
* 推进橙橙好感；
* 积累照护记录；
* 为后续剧情或任务做准备。

每个自由行动按钮需要定义：

* 消耗什么；
* 增加或减少哪些数值；
* 是否影响普通孩子群体数值；
* 是否影响橙橙好感；
* 是否有每日次数限制；
* 是否需要前置条件。

### 两阶段对比

| 维度     | 固定行动阶段             | 自由行动阶段                    |
| ------ | ------------------ | ------------------------- |
| 核心功能   | 给全体孩子安排当天活动        | 玩家在房间内做具体行动               |
| 主要对象   | 普通孩子群体             | 玩家主控、照护所、橙橙               |
| 主要数值   | 群体信任度、群体压力值        | 个人属性、金钱、名望、橙橙好感           |
| 每日次数   | 每天 3 次，必须完成后进入自由行动 | 受剩余行动力限制                  |
| 是否接小游戏 | 第一版只占位，后续可单独模块实现   | 第一版以按钮交互为主                |
| 场景要求   | 只能选择已开放场景          | 只能进入已开放场景                 |
| 行动力消耗  | 按活动强度消耗 1–3 点      | 通常每个按钮消耗 1 点，少数休息按钮消耗 0 点 |

## 2.3 地图：场景候选（修订版）

### 场景 2：绘画室

**开放状态**：初始不开放。第一天即可通过电脑修建，修建后当天立刻可用。

**氛围**：安静，有画架、矮桌、纸张和画具，适合观察孩子通过非语言方式表达。

绘画室可安排全体孩子进行绘画活动。它和橙橙主线关系最强，但固定行动阶段仍然影响普通孩子群体数值。

建议数值：

| 安排次数    | 行动力消耗 | 群体信任度 | 群体压力值 | 说明               |
| ------- | ----: | ----: | ----: | ---------------- |
| 第 1 次   |     2 |    +7 |    -2 | 安静、低刺激，创作空间提高接受度 |
| 第 2 次重复 |     2 |    +4 |    +0 | 仍有收益，但新鲜度下降      |
| 第 3 次重复 |     2 |    +2 |    +3 | 创作疲劳出现，压力略升      |

#### 自由行动阶段可交互按钮

| 按钮          |    消耗 | 效果                        | 每日限制 |
| ----------- | ----: | ------------------------- | ---- |
| 观察孩子们的绘画    | 1 行动力 | 专业理解 +；获得照护记录素材           | 1 次  |
| 整理画作观察      | 1 行动力 | 提高照护记录质量；可能解锁橙橙相关线索       | 1 次  |
| 筹备公益展览 / 拍卖 | 1 行动力 | 根据名望线性获得金钱；群体信任度下降        | 1 次  |
| 与橙橙互动       | 1 行动力 | 橙橙核心互动点；正确选择增加好感，错误选择降低好感 | 1 次  |

说明：
“筹备公益展览 / 拍卖”进入第一版，但不加入额外选项。它默认获得金钱，同时降低群体信任度，体现将孩子作品转化为机构资源时的伦理代价。

公益展览 / 拍卖收益采用线性公式，便于实现。候选公式：

```text id="ehhsqo"
收益 = 基础收益 + 名望 × 系数
```

例如：

```text id="7v0mzl"
收益 = 100 + 名望 × 5
```

具体系数可在试玩后调整。

## 三、 ASD 儿童写作参考研究（书单提炼）

*参考书目：《我想飞进天空》东田直树 / 《Sincerely, Your Autistic Child》/ 《Thinking in Pictures》Temple Grandin / 《A Kind of Spark》Elle McNicoll / 《Uniquely Human》Barry Prizant*

### 3.1 照护所群体活动中可能呈现的 ASD 特征

| 维度 | 在照护所中可能的表现 | 写作与玩法设计提示 |
| :--- | :--- | :--- |
| **沟通方式** | 不说话 ≠ 没有想法。可能用指物、发声、重复词语、图卡或身体动作来传递意思。 | 对话选项不应只有"语言回应"。玩家选择"观察孩子的手指方向"也应作为有效行动。 |
| **感官处理** | 对特定声音、光线、材质、气味极度敏感或渴求。同样的环境对不同孩子可以是放松，也可以是折磨。 | 场景设计需要区分"当前孩子的感官状态"，同一场景在不同敏感度下应有不同效果值。 |
| **对规则与变化的反应** | 依赖可预期的结构。日程被打乱（如临时换活动室）可以引发显著的情绪反应，不是"任性"。 | 突发事件可包含"计划外变更"类型。玩家处理方式影响安全感，而不仅仅是压力数值。 |
| **兴趣与重复行为** | 重复动作（摇晃、排列、翻书）往往是自我调节工具，不是需要被"矫正"的问题行为。 | 玩家不应有"阻止重复行为"的选项。正确做法是理解并判断它在帮孩子应对什么。 |
| **情绪调节** | 情绪崩溃（meltdown）不是发脾气，是感官/认知过载后失去调节能力的结果。 | 崩溃前有预警信号，玩家需要在"压力值爆红"前行动。干预时机比干预方式更关键。 |
| **与同伴/成人互动** | 与同伴互动的方式可能不符合"社交规范"，但未必是拒绝关系。有些孩子非常在意你，只是表达方式不同。 | 信任积累不应设计成"多对话=高信任"。有时"安静陪伴"比主动交流更有效。 |
| **Masking（掩饰）** | 部分孩子（尤其是女孩或高功能谱系）会强迫自己"表现正常"，代价是极大的内耗和崩溃风险。 | 如果橙橙有 masking 倾向，"今天她表现很好"可能是危险信号，而不是正向进展。 |

### 3.2 核心角色橙橙：方向候选

> 设计原则（来自书单）：橙橙不只是"ASD知识的搬运工"，她要有自己的欲望、目标和处境。她的行为是有意义的适应方式，不是症状。

---

**方向 A：画画是她的语言**

*   **外在表现**：很少开口，但会一直画。画里有非常具体、大量的细节（如把每天早餐的碗碟都画出来）。不喜欢被打断。
*   **大人容易误解的地方**：认为她"沉浸自我、不关心他人"。其实她在用画画记录和理解周围发生的事。
*   **她真正在做什么**：在用视觉方式建立世界的秩序感，画画就是她的内部语言。
*   **玩法机制**：玩家可选"看她在画什么"来获取信息或触发对话支线；打断她画画是负分操作；赠送绘本/新画材是高效的信任积累方式。
*   **开发成本**：低（主要是弹窗+图文对话）。
*   **写作风险**：容易写成"神奇天才儿童"（绘画技能被过度浪漫化）。需要保留她的挣扎和内耗，不只展示天赋。

---

**方向 B：规则守护者**

*   **外在表现**：对日程、规则有极强执行欲，若别人不遵守会非常痛苦（但自己不一定总能解释原因）。对新面孔、新活动有明显抵抗。
*   **大人容易误解的地方**：认为她"固执、控制欲强"，实际上规则是她理解这个混乱世界的唯一可靠工具。
*   **她真正在做什么**：用规则对抗不可预测性带来的恐惧，不是想控制别人。
*   **玩法机制**：每天保持日程稳定可累积安全感；突发事件（日程变更）对她影响最大；玩家可通过提前"用视觉日程卡告知变化"来缓冲。
*   **开发成本**：中（需要设计日程稳定/变更的触发逻辑）。
*   **写作风险**：容易把她写成"障碍生成器"——制造麻烦的配件，而不是有内心生活的角色。

---

**方向 C：Masking 女孩**

*   **外在表现**：表面"表现良好"，会说谢谢、会排队、会配合指令，大人评价"进步很大"。但体力消耗极大，回家后经常崩溃（家长视角可见）。
*   **大人容易误解的地方**：以为她"已经适应了"或"基本正常了"，实际上她在持续高强度地伪装。
*   **她真正在做什么**：在强迫自己符合他人期待，代价是不断消耗自己。
*   **玩法机制**："今天橙橙表现很好"可以是预警指标；玩家需要学会辨别真正放松的信号与masking；给她创造不需要表演的空间是关键任务。
*   **开发成本**：高（需要设计双重状态系统：表面状态 vs. 真实状态，玩家需要通过行动解锁后者）。
*   **写作风险**：概念先行风险较高。需要非常细腻的叙事才能让玩家感受到，而不是"被科普"。

---

**方向 D：声音探索者**（保留现有小橙的感官敏感方向）

*   **外在表现**：对特定频率的声音极敏感（如某种金属声），但同时对某些声音有强烈的兴趣和寻求行为（如反复听同一段音乐片段）。
*   **大人容易误解的地方**：认为她"挑剔、怕吵"，但没注意到她也在主动寻求某些特定的感官体验。
*   **她真正在做什么**：在主动辨别和筛选感官世界——她有自己的感官偏好，只是与大多数人的不同。
*   **玩法机制**：安静角落和降噪耳机对她最有效；在特定音乐/声响环境下触发特殊互动；感官工具（耳机、柔软材质玩具）是与她建立信任的钥匙。
*   **开发成本**：低（现有代码已有基础架构，继承性最强）。
*   **写作风险**：容易停留在"她怕吵"这一表面层，忽略她对特定声音的主动寻求这一更复杂的面向。

## 四、 世界观转化为玩法的核心映射总结

1.  **感官差异 → 场景被动**：橙橙对声音过载需要"降噪耳机"和"安静角落"支持；规律依赖孩子需要"视觉日程卡"等结构化支持。
2.  **照护困境 → 资源取舍**：有限精力、紧缺资金、波动的口碑。在关注孩子个别发展（提信任、降压力）与处理机构外联行政（赚资金、争口碑）之间保持动态张力。
3.  **行为有意义 → 玩法选项设计**：玩家不应有"纠正/阻止行为"的选项，而应选择"这个行为在帮孩子应对什么"，并以此做出回应。这是本游戏与一般养成游戏最核心的差异。
4.  **结局评价 → 机构与孩子双轨**：大结局不仅考核"照护所是否活下来"（资金/口碑），更考核"孩子是否在这里找到了安全感"（信任度/安全感），落实"以理解代替治愈"的公益初心。
