# GAMEPLAY_PLAN_V2.md — 星光照护所·玩法设计 V2

> 方向：主线剧情驱动 + 属性养成 + 角色线解锁，叙事节奏+模拟经营压力结构。
> 实现约束：`src/main.js` + `src/style.css`，Vite + Phaser，纯前端，localStorage 存档。

## 1. 游戏类型定位

**文字驱动的经营养成公益游戏**，兼具两种层次的体验：

| 层次 | 体验重点 |
|---|---|
| 宏观（机构经营） | 每天管理有限资源，应对照护所的生存压力——资金、家长信任、人手 |
| 微观（角色线） | 通过日常互动积累孩子信任，在正确时机触发深层剧情，理解孩子的内在世界 |

两个层次之间存在资源张力：**专注孩子会消耗机构资源，忽视孩子会影响角色线进展**。这个张力是核心乐趣所在，也是公益主题的核心表达——照护不是一件可以"兼顾所有"的事。

## 2. 核心玩法

### 2.1 基本节奏

每天分为三个阶段，依次推进，不可倒退：

```
[晨会阶段]  → 主任简报当日状况，宣布当日任务和突发事件
     ↓
[行动阶段]  → 玩家在办公室 Hub 点击行动按钮，消耗精力/资金处理事务
     ↓
[收场阶段]  → 点击"结束今天"，触发日结算，检测当日任务是否完成
              ├── 任务通过 → 自动存档，进入下一天晨会
              └── 任务失败 → 触发当天的失败结局文字，提示是否重试当天
```

### 2.2 行动的性质

行动分为两类，每天精力有限，两类都要兼顾：

| 行动类型 | 举例 | 消耗 | 效果 |
|---|---|---|---|
| **机构行动** | 整理档案、接待家长、联系赞助方、处理投诉 | 精力 1–2 | 影响资金、家长信任、口碑 |
| **孩子互动** | 陪伴孩子、引导活动、处理突发情绪 | 精力 1–2 | 影响孩子压力、信任、安全感 |

同一行动每天只能执行**有限次数**（通常 1 次），避免玩家无限堆叠单一策略。

### 2.3 突发事件

每天行动阶段中途可能插入 1–2 个突发事件（弹窗形式）：
- 不消耗精力，但要求玩家做选择
- 正确选择维持或改善数值；错误选择扣减数值，并可能影响次日任务难度
- Day 1 固定触发 1 个突发事件（小橙的声音事件），让玩家初步感受 ASD 孩子的需求

## 3. 主线剧情与角色线的关系

### 3.1 主线：照护所的生存危机

主线不围绕单个孩子，而是**照护所本身的经营压力**。每天都有机构层面的问题需要解决，形成时间压迫感：

```
Day 1 — 新人第一天，主任交代现状：资金紧张，有家长正在考虑是否退出
Day 2 — 一位家长打来电话，对某次照护处理方式表示不满
Day 3 — 机构季度检查即将到来，需要提前准备档案
Day 4 — 赞助方来访，口碑直接影响赞助金额
Day 5 — 家长公开日，孩子状态被家长观察到，影响信任
...
```

主线驱动节奏，提供每天的**硬性任务**（通过/失败）。

### 3.2 角色线：孩子的个人故事

角色线通过**信任值积累 + 关键事件触发**解锁，不在主线上强制推进：

```
信任 0–29   → 孩子对你戒备，互动选项有限，只有基础陪伴
信任 30–59  → 孩子开始接受你，解锁更多互动选项，偶尔有简短的个人片段
信任 60–89  → 孩子愿意向你展示他们真实的一面，触发深层剧情（回忆/独白）
信任 90–100 → 孩子线阶段性结局：一段文字描述孩子"找到了更安全的方式表达自己"
```

角色线不影响机构主线的通过/失败，但**角色线进展会影响大结局的文字内容**。

### 3.3 两条线的互动关系

- 在机构危机最严重的那天（如资金危机），**过度关注孩子会导致机构任务失败**——这是刻意设计的困境，没有完美解法，只有取舍。
- 孩子信任度足够高时，在家长公开日的表现会更稳定，反过来**提升机构口碑**——让玩家感受到耐心与信任是有回报的，但不是立竿见影的。

## 4. 教师办公室 Hub 及其他场景

### 4.1 主 Hub：教师办公室

每天行动阶段的主要界面。不需要玩家走路，而是**点击屏幕上的功能区域或按钮发起行动**。

```
┌──────────────────────────────────────────────────────────┐
│  [顶部 HUD]  第 N 天 | 精力: ●●●○○ | 资金: ¥XXXX | 口碑: XX | 家长信任: XX  │
├──────────────────────────────────────────────────────────┤
│                                                          │
│   ┌─────────────┐   ┌──────────────────┐                │
│   │  主任（可点）│   │   今日任务栏      │                │
│   │  打开晨会   │   │  ☐ 主线任务       │                │
│   │  / 求建议   │   │  ☐ 可选任务       │                │
│   └─────────────┘   └──────────────────┘                │
│                                                          │
│   ─────────── 行动区域 ─────────────────                 │
│   [ 前往活动室 → 照顾孩子 ]   [ 处理文件 → 机构行政 ]   │
│   [ 联系赞助方 ]              [ 接待家长来访 ]           │
│   [ 使用感统室 ]              [ 学习 ASD 知识 ]          │
│                                                          │
│   [结束今天] （确认后触发结算）                           │
└──────────────────────────────────────────────────────────┘
```

**每个行动按钮**点击后弹出：
1. 简短的情境描述（1–3 行文字）
2. 若有选择：2–4 个选项，各有不同的数值效果
3. 若无选择：单一结果 + "确认"按钮

### 4.2 孩子互动子界面

点击"前往活动室"后，进入孩子互动子界面。界面形式类似橙光的对话场景：

```
┌──────────────────────────────────────────────────────────┐
│  [当前孩子列表：小橙 | 小木 | 小雨]                       │
├──────────────────────────────────────────────────────────┤
│                                                          │
│   [孩子名称]  当前状态：平静 / 焦虑 / 紧绷               │
│   压力: ████░░  信任: ███░░░                             │
│                                                          │
│   [今天的情境描述文字……]                                  │
│                                                          │
│   [ 选项 A ]   [ 选项 B ]   [ 选项 C ]                  │
│                                                          │
│   [← 返回办公室]                                         │
└──────────────────────────────────────────────────────────┘
```

**每名孩子每天**：
- 固定可互动 1 次（主要行动）
- 如当日有突发事件涉及该孩子，额外触发 1 次选择

### 4.3 晨会场景（每天开始自动触发）

全屏文字场景，主任简短交代：
- 昨天的结果复盘（1–2 句）
- 今日的机构状况（资金/家长信任变化）
- 今日任务说明
- 可能有的特殊预告

节奏参考橙光的章节开场。点击"开始今天"进入行动阶段。

### 4.4 日结算场景（点击"结束今天"后触发）

全屏文字场景：
- 当日行动汇总（做了什么，数值变化）
- 当日任务检测结果
- 任务通过：简短正向描述 + 自动存档 + "进入明天"按钮
- 任务失败：失败情境描述（不说教，说明后果）+ "重试今天" / "继续（不重试）"按钮

## 5. 核心资源与属性设计

### 5.1 机构资源（每天结算）

| 资源 | 范围 | 归零/上限后果 |
|---|---|---|
| **精力** | 0–8/天 | 耗尽后无法发起任何行动，强制结束当天 |
| **资金** | 数值型 | 归零触发"机构运营危机"事件，主线失败 |
| **家长信任** | 0–100 | 跌破 20 触发家长集体投诉，主线失败 |
| **口碑** | 0–100 | 影响赞助金额和家长信任的恢复速度 |

每天固定支出：精力全部重置；资金扣除日常运营成本（工资、耗材），具体数额由当天事件决定。

### 5.2 孩子属性（持久积累，跨天保留）

每名孩子独立维护以下属性：

| 属性 | 范围 | 说明 |
|---|---|---|
| **压力** | 0–100 | 当天敏感，高压触发情绪事件；每天自然回落少量 |
| **安全感** | 0–100 | 慢速积累，影响互动选项数量 |
| **信任** | 0–100 | 核心养成数值，积累解锁角色线剧情 |
| **状态** | 平静/焦虑/紧绷 | 由压力值驱动，影响当天互动文本基调 |

### 5.3 数值张力设计

以下几对数值之间存在刻意的张力，避免最优解太明显：

| 张力 | 具体体现 |
|---|---|
| 精力 vs 孩子互动 | 花精力照顾孩子，就没精力处理机构行政 |
| 孩子信任 vs 机构口碑 | 某些情况下，保护孩子不被参观会损失口碑 |
| 当下压力 vs 长期信任 | 强行让孩子"表现好"可降低当日压力，但伤害信任 |
| 资金 vs 设施 | 购置感统器材能长期降低孩子压力，但消耗一次性资金 |

## 6. 每日流程（通用结构）

```
[晨会]
  主任简报 → 宣布今日任务（1 个硬性 + 0–1 个可选）→ 点击"开始今天"

[行动阶段]  精力 = 8，玩家自由分配
  ├── 机构行动（1–4 项可选）
  │     每项消耗 1–2 精力，处理文件/家长/赞助等
  ├── 孩子互动（每名孩子各 1 次）
  │     消耗 1 精力，进入孩子子界面选择处理方式
  ├── 突发事件（自动插入，0–2 次）
  │     弹窗选择，不消耗精力，但影响数值和次日难度
  └── [结束今天] 按钮（任意时刻可点，精力耗尽后强制弹出）

[日结算]
  资金结算 → 家长信任微调 → 任务检测 → 显示结果 → 存档或重试
```

## 7. Day 1 最小可玩性设计

Day 1 是玩家的入职第一天，核心目标：**让玩家在一次完整流程中理解游戏的所有基本机制**。

### 7.1 晨会内容

主任说明：
- 照护所基本情况（资金状况、三名孩子的基本特征）
- 今日任务：**接待一位来访家长（陈先生，小橙的父亲）**
- 操作提示内嵌在主任台词中，不做独立教程弹窗

### 7.2 可用行动（Day 1）

| 行动 | 精力消耗 | 说明 |
|---|---|---|
| 去活动室陪孩子 | 1/次，每名孩子 1 次/天 | 进入孩子子界面 |
| 使用感统室 | 1 | 可降低所有孩子压力 5 点 |
| 整理入职档案 | 1 | 机构行动，口碑 +3（新员工加分） |
| 接待来访家长 | 2 | **今日硬性任务**，见 7.3 |
| 向主任请教 | 1 | 获取一条 ASD 知识提示（不影响数值） |

### 7.3 今日硬性任务：接待小橙父亲

- 触发时机：玩家点击"接待来访家长"行动按钮
- 情境：小橙的父亲陈先生来访，想了解孩子最近的状态；他情绪偏低落，言语间透露出疲惫和自责
- 玩家面对 3 个选项：
  - **A. 如实说明小橙目前适应状况（中性）**：家长信任 +5，资金不变；陈先生离开时点了点头，神情没有明显变化
  - **B. 重点说明小橙今天的积极时刻（偏正向）**：家长信任 +10，口碑 +3；陈先生眼睛亮了一下，"谢谢你告诉我这个。"
  - **C. 建议陈先生减少探访频率（操作性）**：家长信任 -10；陈先生沉默了一会儿，"我知道了。"（离开）

> 设计意图：三个选项没有"错误答案"，只有不同的价值判断——诚实、正向引导、还是专业建议。让玩家感受到照护工作中沟通的复杂性。

### 7.4 Day 1 突发事件：小橙的声音事件

- 触发时机：玩家第一次进入活动室与小橙互动，或在孩子子界面停留超过一定操作次数后自动弹出
- 内容：走廊传来关门声，小橙捂住耳朵，身体蜷缩
- 选项：
  - **A. 移到安静角落，轻声告知"快结束了"**（正确）：压力 -15，信任 +10
  - **B. 要求他"别这样，大家都在看"**（错误）：压力 +20，信任 -15，家长信任 -5（若当日家长已在场）

> 此事件是 Day 1 最重要的公益教育时刻。无论玩家选 A 还是 B，弹出后都附有 1 行**无说教性的说明**文字。

### 7.5 Day 1 可选收获

- 如果玩家与小木互动（不打扰 / 安静观察），可触发小木的简短日常片段（1 段文字）
- 如果玩家与小雨互动（一起看窗外），小雨信任 +8，当天状态变为"平静"

## 8. Day 1 任务检测与失败条件

### 8.1 硬性任务

| 任务 | 通过条件 | 失败条件 |
|---|---|---|
| 接待小橙父亲（陈先生） | 玩家在精力耗尽前点击"接待来访家长"并完成选择 | 精力耗尽前未触发该行动 |

> 精力耗尽 = 强制结束当天，若任务未完成则进入失败判定。

### 8.2 失败结局文字（Day 1）

```
陈先生在等候区等了很久，没有人来接待他。
他最后起身，轻声跟前台说了一句"没关系"，然后离开了。
门关上的声音，比平时更轻。

[家长信任 -20]

— 你可以重试今天，或选择继续。
```

> 设计意图：失败后果落在机构层面（家长信任），而非评判玩家"不关心孩子"。选择"继续而不重试"是被允许的，某些玩家可能想看完失败路线的叙事。

### 8.3 可选任务（不影响通过/失败）

| 任务 | 完成奖励 |
|---|---|
| 与三名孩子各互动 1 次 | 次日晨会主任对你的评价提升（口碑 +3） |
| 向主任请教 1 次 | 解锁"ASD 知识卡片"——小橙为何对声音敏感的科普内容 |

## 附：新旧方案对比

| 维度 | V1（地图移动版） | V2（Hub 点击版）|
|---|---|---|
| 主要操作 | WASD 移动 + 靠近互动 | 点击行动按钮 |
| 叙事密度 | 较低（靠近才触发） | 较高（晨会/结算都有文字） |
| 机构经营感 | 弱（无每日任务） | 强（任务通过/失败） |
| 角色线 | 线性（只有小橙主线） | 开放（三人均可积累，分阶段解锁） |
| 失败压力 | 无（只有阶段性结尾） | 有（每日硬性任务可失败，可重试） |
| 夜晚梦境 | 核心玩法之一 | 搁置，作后续扩展 |
| 实现难度 | 中（需要碰撞/移动系统） | 低（纯 UI 交互，无需移动） |
